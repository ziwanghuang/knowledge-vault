## 二、面试中最可能被追问的重点领域

根据面试官的真实追问模式（技术选型 WHY → 规模极限 → 故障场景 → 设计权衡 → 线上实战），以下是三个项目各自最可能被重点追问的 Top 5：

### data-platform Top 5 追问热点
1. **"先 Kafka 后 DB"的一致性设计** — 这是最独特的设计决策，面试官 100% 会追问
2. **6000 Agent 规模下的性能瓶颈** — "你说优化了 10 倍，数据从哪来"
3. **Redis 分布式锁 vs etcd 选举** — 脑裂场景、主从切换期间锁失效
4. **微服务拆分的边界和通信模式** — 为什么拆 6 个、为什么这样划分
5. **Kafka 和轮询的取舍** — "500 节点以下值不值得引入 Kafka"

### distributed_task_platform Top 5 追问热点
1. **Prometheus 驱动的智能选节点** — 这是最大的差异化亮点，必深挖
2. **乐观锁抢占 vs 分布式锁** — 为什么不同场景用不同方案
3. **补偿器 V1→V2 的演进** — 分库分表后的分片巡检机制
4. **Snowflake ID 的已知 Bug** — 位溢出、序列号不绑定毫秒（主动暴露缺陷反而加分）
5. **DAG 引擎的 ANTLR4 DSL** — 怎么编译、怎么执行、环路检测

### ai-dataplatform-ops Top 5 追问热点
1. **LangGraph vs 其他框架的选型** — CrewAI/AutoGen/MetaGPT 对比
2. **RAG 的三路检索 + RRF 融合** — Dense/Sparse/Graph 各自的价值
3. **MCP 协议的设计决策** — 为什么不直接用 gRPC、为什么 Go 不 Python
4. **LLM 输出不确定性的工程化处理** — instructor、retry、降级、幻觉检测
5. **showcase vs production 的差距** — "你说这是 showcase 级别，那你怎么证明架构是生产级的"

## 三、🔴 当前文档未覆盖的关键盲区

### 盲区 1：Go 语言基础知识追问（三项目共有）

**风险等级：🔴 高**

面试官看到你用 Go 写了两个项目，大概率会追问 Go 语言本身的知识。当前文档只覆盖了项目内的 Go 使用，没有覆盖：

| 未覆盖的 Go 追问 | 示例问题 |
|-----------------|---------|
| **goroutine 调度模型（GMP）** | "Go 的 goroutine 和 OS 线程什么关系？GMP 模型讲讲" |
| **channel 底层实现** | "有缓冲和无缓冲 channel 的区别？底层数据结构？" |
| **GC 机制** | "Go 的 GC 算法？三色标记法？STW 时间如何优化？" |
| **内存模型与 happens-before** | "goroutine 之间的内存可见性怎么保证？" |
| **defer/panic/recover 机制** | "defer 的执行顺序？多个 defer 怎么排列？" |
| **sync 包深度** | "sync.Map 的实现原理？什么场景适用？" |
| **interface 底层** | "空 interface 和非空 interface 的区别？iface vs eface？" |
| **slice/map 底层** | "slice 扩容策略？map 的 hash 冲突解决？" |
| **context 原理** | "context 的取消传播机制？WithCancel/WithTimeout 的区别？" |

**建议**：补一篇 **Go 高频八股文** 文档，15-20 题，结合项目中的实际使用场景回答。

### 盲区 2：MySQL/Redis 基础知识追问（三项目共有）

data-platform 的 13 篇（互联网高频场景题）覆盖了一些，但 distributed_task 和 ai-ops 的文档中没有独立的基础知识追问准备。面试官可能问：

| 未覆盖的追问 | 关联项目 |

**建议**：不需要单独补文档，但面试前要确保能在这些问题上流畅回答，可以引用 data-platform 的 13 篇作为统一复习材料。
